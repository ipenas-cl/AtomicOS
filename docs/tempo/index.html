Tempo Programming Language

A deterministic systems programming language being developed for AtomicOS security modules. Currently in early development with basic compilation capabilities.

Current Implementation Status

✅ Working Features
  • Basic function definitions
  • Simple return statements
  • Integer literals
  • x86 assembly code generation
  • Command-line compiler interface
⚠️ Partial Implementation
  • Variable declarations (let statements)
  • Function parameters
  • Basic arithmetic expressions
  • Type system (int32 only)
❌ Not Yet Implemented
  • Complex expressions and operators
  • Control flow (if/else, loops)
  • Function calls
  • WCET analysis
  • Memory safety checks
  • Formal verification support

Working Example

The following simple Tempo program compiles successfully:

// Simple function that returns a constant function test(): int32 { return 42 }

Compiles to x86 assembly:

section .text bits 32 global test test: push ebp mov ebp, esp mov eax, 42 ret pop ebp ret

Usage:

$ cd tools $ ./tempo_compiler input.tempo output.s === Tempo Compiler v2.0 === Compiling: input.tempo Lexer: 11 tokens Parser: success Code generation: success Output written to: output.s

Development Roadmap

Phase 1: Basic Compiler Infrastructure

Lexer, basic parser, simple AST, and x86 code generation for functions and return statements.

Phase 2: Core Language Features

Complete expression parsing, function parameters, local variables, and arithmetic operations.

Phase 3: Control Flow

If/else statements, loops, function calls, and proper variable scoping.

Phase 4: Deterministic Features

WCET analysis, memory bounds checking, and deterministic execution guarantees.

Phase 5: Security Integration

AtomicOS security module compilation, formal verification support, and production deployment.
