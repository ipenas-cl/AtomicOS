// Constant-time cryptographic operations in Tempo
@constant_time
@wcet(1500)
function crypto_compare(a: ptr<int8>, b: ptr<int8>, len: int32) -> bool {
    let result: int32 = 0;
    
    // Compare all bytes regardless of match (no early exit)
    for let i = 0; i < len; i = i + 1 {
        result = result | (a[i] ^ b[i]);
    }
    
    // Constant-time conversion to boolean
    return ct_select(result == 0, true, false);
}

@wcet(500)
function secure_task() -> void {
    with security_level(KERNEL) {
        // Guaranteed to complete in 500 cycles
        process_critical_data();
    }
}

Why Tempo?

⏱️

WCET Analysis

Every function has a guaranteed worst-case execution time. Perfect for real-time systems where timing matters.

🔐

Security by Design

Built-in security levels, constant-time operations, and memory safety. No buffer overflows, no timing attacks.

🚀

Zero-Cost Abstractions

High-level language features compile to efficient machine code. No runtime overhead, no garbage collector.

🎯

Deterministic

Same input always produces same output. No hidden allocations, no unpredictable behavior.

🔧

Systems Programming

Direct hardware access, inline assembly, and low-level control when you need it.

📚

Modern Syntax

Clean, readable syntax inspired by the best of modern languages. Easy to learn, powerful to use.
